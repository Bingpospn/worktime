# 工时统计 · 网页版（PWA）

可添加到 iPhone 主屏幕、像原生 App 一样全屏运行、离线可用的工时统计网页应用。日历宫格视图，点某天即可录入打卡/请假，自动按公司规则算有效工时并跟踪本月 8 小时达标。

内置 **2026 年中国法定节假日与调休补班日**（国办发明电〔2025〕7 号）。

---

## 放到 iPhone 上（两步）

### 第 1 步：把这个文件夹放到网上（任选一种，免费）

> 需要 HTTPS 网址才能离线和"添加到主屏幕"。下面两种都自带 HTTPS。

**方式 A · Netlify Drop（最快，可不注册先试）**
1. iPhone 或电脑浏览器打开 `https://app.netlify.com/drop`
2. 把整个 `PWA` 文件夹拖进页面（手机可用"上传文件夹"）
3. 立刻得到一个 `https://xxx.netlify.app` 网址（想长期保留就免费注册登录）

**方式 B · GitHub Pages**
1. 新建一个仓库，上传 `PWA` 文件夹内的全部文件（`index.html` 要在仓库根目录或 `/docs`）
2. 仓库 Settings → Pages → 选择分支，保存
3. 得到 `https://你的用户名.github.io/仓库名/` 网址

### 第 2 步：在 iPhone 上添加到主屏幕
1. 用 **Safari** 打开上一步的网址
2. 点底部"分享"按钮 → **添加到主屏幕** → 添加
3. 主屏出现"工时统计"图标，点开即全屏运行，断网也能用

## 使用

- 顶部 **‹ ›** 切换月份，"今天"回到本月。
- 时间精确到**秒**：时、分用系统时间选择器，**秒在右侧小框单独填**（iOS 原生选择器只到分钟，故拆开处理）；默认 09:30:00，可用 8:30 / 18:00 / 套用最优下班 快速填。
- 日历格子颜色：绿色=达标≥8h（含节假日自动 8h）、橙色=不足8h、灰色=请假、蓝色"班"=调休补班、淡色=休息日；今天有蓝框。
- **落在周一~周五的放假日（含调休借放日）自动按 8h 计入，无需打卡**。
- 录入上班时间时，按当月进度**实时算出"今日最优下班时间"**；每周最后两个工作日会加重提醒，避免月均掉到 8h 以下。
- 顶部卡片显示已完成/目标、应计·已完成·剩余与智能提示；底部是本月每日工时柱状图（含 8h 基准线，点柱可跳到当天）。
- 点卡片右上角**"设置月薪"**填入税前月薪（仅存本机、不上传）。薪资与缺勤规则：
  - **计薪天数 = 当月工作日（含请假当天）+ 法定节假日**；法定节假日期间的工作日正常计薪。
  - **请假**：整天无薪，按 **7h 缺勤（0.875 天，从 9:30 起算）**；**迟到**：按"取整分钟 ÷ 480"折算缺勤。
  - **缺勤扣款 = 月薪 ÷ 计薪天数 × 缺勤天数（迟到＋请假）**；编辑页显示当次扣款，月度卡片显示合计。
  - **月末**（当月最后 5 天或往月）显示 **预计实发（税后）= 剩余额 − 应纳税额**，其中 剩余额 = 月薪 − 缺勤扣款、应纳税额 =（剩余额×80%）×20%，即 实发 ≈ 0.84 × 剩余额。仅为按你提供口径的估算，非税务意见。

## 数据与注意事项

- 数据存在浏览器本地（localStorage），**与网址绑定**。请始终用同一个网址（或主屏图标）打开，数据会一直保留。
- 在 Safari 里"清除网站数据"或更换托管网址会导致历史清空；换网址前可先记录。
- 必须用 **HTTPS** 网址才能离线运行与安装；用 `http://` 的本机地址只能在线用，不能离线。
- 节假日自动 8h 的判定：脚本里 `HOL` 是全部放假日，凡落在**周一~周五**的（含调休借放日）都自动按 8h 计入；`MK` 是调休补班日（需正常上班）。目标天数与已完成都含节假日，净额为 0，不改变你在工作日要补的工时。
- 迟到折算：晚于 9:30 的时长不足 1 分钟向上进 1（如 31′20″ 记 32′），折算无薪请假天数 = 取整分钟 ÷ 60 ÷ 8，显示到小数点后 5 位。
- 历史数据自动兼容：旧版本按分钟存的记录会在打开时自动转换为秒，不会丢。
- 每次更新代码后，手机上的 App 可能需**关掉再重新打开一次**才会加载新版本（离线缓存已升级，会自动刷新）。
- 节假日数据目前仅含 **2026 年**。跨年时更新 `index.html` 顶部脚本里的 `HOL`（放假日）与 `MK`（调休补班日）即可。
